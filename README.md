# TrendPullbackEA - Expert Advisor для MetaTrader 5

## 📖 Описание

**TrendPullbackEA** - это автоматический торговый робот для платформы MetaTrader 5, который торгует по тренду с входом на откатах. Робот использует комбинацию технических индикаторов для определения тренда и поиска оптимальных точек входа.

### Стратегия торговли

1. **Определение тренда**: Использует EMA200 на часовом таймфрейме (H1) для определения общего направления тренда
2. **Поиск откатов**: Анализирует RSI на 15-минутном таймфрейме (M15) для поиска точек входа на откатах
3. **Управление рисками**: Автоматически рассчитывает размер лота на основе процента риска от баланса
4. **Защита прибыли**: Поддерживает трейлинг стоп для защиты прибыли

## 📦 Установка

> 📖 **Подробная инструкция по установке** см. в файле [INSTALLATION.md](INSTALLATION.md)

### Краткая инструкция:

1. Скопируйте **все файлы** в папку `MQL5/Experts/TrendPullbackEA/` вашего MetaTrader 5
2. Откройте MetaEditor и скомпилируйте файл `TrendPullbackEA.mq5` (F7)
3. Перетащите робота из папки "Navigator" → "Expert Advisors" на график нужного инструмента
4. Настройте параметры (см. [PARAMETERS_EXAMPLES.md](PARAMETERS_EXAMPLES.md))

## ⚙️ Параметры настройки

### Основные параметры

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `RiskPercent` | Процент риска от баланса на одну сделку | 1.0 |
| `EMAPeriod` | Период EMA для определения тренда | 200 |
| `RSIPeriod` | Период индикатора RSI | 14 |
| `RR` | Соотношение риск/прибыль (Risk/Reward) | 2.0 |
| `ATRPeriod` | Период индикатора ATR | 14 |
| `ATRMultiplierSL` | Множитель ATR для расчета стоп-лосса | 1.5 |
| `MaxSpreadPoints` | Максимально допустимый спред в пунктах | 30 |
| `MagicNumber` | Уникальный номер для идентификации ордеров | 777 |

### Дополнительные параметры

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `UseTimeFilter` | Использовать фильтр по времени торговли | false |
| `StartHour` | Начало торговли (0-23) | 0 |
| `EndHour` | Конец торговли (0-23) | 23 |
| `UseTrailingStop` | Использовать трейлинг стоп | false |
| `TrailingStopATR` | Множитель ATR для трейлинг стопа | 1.0 |
| `TrailingStepATR` | Шаг трейлинг стопа в ATR | 0.5 |
| `EnableLogging` | Включить логирование операций | true |
| `ShowInfoPanel` | Показывать информационную панель на графике | true |

### 🛡️ Консервативная настройка

**Рекомендуется для:** Начинающих трейдеров, небольших счетов, торговли на реальном счете

```mql5
RiskPercent      = 0.5   // Риск 0.5% на сделку
EMAPeriod        = 200   // Стандартный период EMA
RSIPeriod        = 14     // Стандартный период RSI
RR               = 2.5    // Высокое соотношение риск/прибыль (2.5:1)
ATRPeriod        = 14     // Стандартный период ATR
ATRMultiplierSL  = 2.0    // Больший стоп-лосс для безопасности
MaxSpreadPoints  = 20     // Строгий контроль спреда
UseTimeFilter    = true   // Торговля только в определенное время
StartHour        = 8      // Начало торговли в 8:00
EndHour          = 18     // Конец торговли в 18:00
UseTrailingStop  = true   // Включен трейлинг стоп
TrailingStopATR  = 1.5    // Консервативный трейлинг стоп
TrailingStepATR  = 0.5    // Малый шаг трейлинг стопа
EnableLogging    = true   // Логирование включено
```

**Характеристики:**
- Низкий риск на сделку (0.5%)
- Высокое соотношение риск/прибыль (2.5:1)
- Больший стоп-лосс для защиты от ложных сигналов
- Торговля только в определенные часы
- Меньше сделок, но выше качество

## 📈 Управление рисками

Робот автоматически рассчитывает размер лота на основе:
- Процента риска от баланса (`RiskPercent`)
- Расстояния до стоп-лосса
- Характеристик инструмента (стоимость тика, размер тика)

**Формула расчета лота:**
```
Лот = (Баланс × RiskPercent / 100) / (Расстояние_стоп_лосса × Стоимость_тика)
```

Это гарантирует, что при срабатывании стоп-лосса вы потеряете ровно указанный процент от баланса.

1. **Тестирование**: Всегда тестируйте робота на демо-счете перед использованием на реальном счете

## 🐛 Устранение неполадок

### Робот не открывает позиции:
- Проверьте, что спред не превышает `MaxSpreadPoints`
- Убедитесь, что условия входа выполнены (тренд + RSI)
- Проверьте логи на наличие ошибок

### Робот открывает слишком много/мало позиций:
- Отрегулируйте `RSIPeriod` (меньше = больше сигналов)
- Измените пороги RSI (для покупки < 40, для продажи > 60)
- Используйте фильтр времени для ограничения активности

### Стоп-лосс слишком большой/маленький:
- Измените `ATRMultiplierSL` (больше = больший стоп-лосс)
- Проверьте значение ATR для вашего инструмента
