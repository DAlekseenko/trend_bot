## 📊 Логическая структура

### 1. Инициализация (OnInit)
- Создание индикаторов (EMA, RSI, ATR)
- Проверка успешности создания
- Настройка торгового объекта

### 2. Основной цикл (OnTick)
- Управление открытыми позициями (трейлинг стоп)
- Проверка нового бара
- Проверка фильтров (время, спред)
- Проверка открытых позиций
- Определение тренда
- Получение значений индикаторов
- Генерация сигнала
- Расчет размера лота
- Открытие позиции

### 3. Модули

#### TrendPullbackEA_Inputs.mqh
Содержит все параметры, которые можно настроить:
- Основные параметры стратегии
- Дополнительные параметры (фильтры, трейлинг стоп)

#### TrendPullbackEA_Types.mqh
Определяет типы данных:
- `TrendDirection` - перечисление направления тренда
- `TradeSignal` - структура торгового сигнала

#### TrendPullbackEA_Trend.mqh
Функции определения тренда:
- `GetTrend()` - определение направления тренда по EMA

#### TrendPullbackEA_Signals.mqh
Генерация сигналов и работа с индикаторами:
- `GenerateSignal()` - создание торгового сигнала
- `GetRSI()` - получение значения RSI
- `GetATR()` - получение значения ATR

#### TrendPullbackEA_Risk.mqh
Управление рисками:
- `CalculateLot()` - расчет размера лота на основе риска

#### TrendPullbackEA_Trade.mqh
Исполнение сделок:
- `ExecuteTrade()` - открытие ордера
- `HasOpenPosition()` - проверка открытых позиций
- `ManageTrailingStop()` - управление трейлинг стопом

#### TrendPullbackEA_Helpers.mqh
Вспомогательные функции:
- `IsSpreadValid()` - проверка спреда
- `IsNewBar()` - проверка нового бара
- `IsTimeToTrade()` - проверка времени торговли

## 🎯 Преимущества модульной структуры

1. **Читаемость** - каждый модуль отвечает за свою область
2. **Поддерживаемость** - легко найти и изменить нужную функцию
3. **Переиспользование** - модули можно использовать в других роботах
4. **Тестирование** - легче тестировать отдельные модули
5. **Масштабируемость** - легко добавлять новые функции

## 📝 Как добавить новый функционал

### Пример: Добавление нового индикатора

1. Добавьте параметры в `TrendPullbackEA_Inputs.mqh`
2. Создайте хэндл в `TrendPullbackEA.mq5` (OnInit)
3. Добавьте функцию получения значения в `TrendPullbackEA_Signals.mqh`
4. Используйте в `GenerateSignal()` или создайте отдельную функцию

### Пример: Добавление нового фильтра

1. Добавьте параметры в `TrendPullbackEA_Inputs.mqh`
2. Создайте функцию проверки в `TrendPullbackEA_Helpers.mqh`
3. Добавьте проверку в `OnTick()` в `TrendPullbackEA.mq5`
