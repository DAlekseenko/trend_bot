## 💰 Рекомендуемые параметры для депозита $5000

### Оптимальная настройка:

```mql5
// УПРАВЛЕНИЕ РИСКАМИ
RiskPercent      = 0.5   // Риск 0.5% на сделку = $25 на сделку
RR               = 3.0    // Высокое соотношение риск/прибыль (3:1)

// ФИЛЬТРЫ КАЧЕСТВА

RSIPeriod        = 14     // Стандартный период RSI
MaxSpreadPoints  = 20     // Строгий контроль спреда (только качественные входы)

// ЗАЩИТА
UseTimeFilter    = true   // Торговля только в определенное время
StartHour        = 8       // Начало европейской сессии
EndHour          = 18      // Конец европейской сессии
UseTrailingStop  = true   // Обязательно включен трейлинг стоп
TrailingStopATR  = 1.5    // Консервативный трейлинг стоп
TrailingStepATR  = 0.5    // Малый шаг

// МОНИТОРИНГ

```

- **Процент прибыльных:** 65-75%

- **Максимальная просадка:** до 5-8% от депозита

## 🔧 Улучшения для стабильной торговли

### 1. Фильтр силы тренда

**Проблема:** Текущий код определяет тренд только по положению цены относительно EMA.

**Решение:** Добавить проверку наклона EMA и расстояния цены от EMA.

```mql5
// Добавить в TrendPullbackEA_Inputs.mqh
input double MinTrendStrength = 0.0001; // Минимальная сила тренда (наклон EMA)
input double MinPriceDistance = 50;     // Минимальное расстояние цены от EMA в пунктах
```

### 2. Фильтр волатильности

**Проблема:** Робот может торговать при слишком низкой или слишком высокой волатильности.

**Решение:** Добавить проверку ATR на минимальный и максимальный пороги.

```mql5
// Добавить в TrendPullbackEA_Inputs.mqh
input bool   UseVolatilityFilter = true;  // Использовать фильтр волатильности
input double MinATRMultiplier    = 0.5;   // Минимальный ATR (от среднего)
input double MaxATRMultiplier    = 2.0;   // Максимальный ATR (от среднего)
```

### 3. Защита от серии убытков

**Проблема:** После нескольких убытков подряд робот продолжает торговать с тем же риском.

**Решение:** Временная остановка торговли после серии убытков.

```mql5
// Добавить в TrendPullbackEA_Inputs.mqh
input bool   UseLossProtection   = true;  // Защита от серии убытков
input int    MaxConsecutiveLosses = 3;     // Максимальное количество убытков подряд
input int    PauseAfterLossHours  = 24;    // Пауза в часах после серии убытков
```

### 4. Break-even стоп

**Проблема:** Прибыльные позиции могут превратиться в убыточные.

**Решение:** Автоматический перевод стоп-лосса в безубыток при достижении определенной прибыли.

```mql5
// Добавить в TrendPullbackEA_Inputs.mqh
input bool   UseBreakEven        = true;  // Использовать break-even стоп
input double BreakEvenRR         = 1.0;   // При достижении прибыли 1x от риска - в безубыток
```

### 5. Фильтр по дням недели

**Проблема:** Некоторые дни недели менее предсказуемы (понедельник, пятница).

**Решение:** Возможность исключить определенные дни из торговли.

```mql5
// Добавить в TrendPullbackEA_Inputs.mqh
input bool   TradeMonday  = true;  // Торговля в понедельник
input bool   TradeFriday  = false; // Торговля в пятницу (обычно исключают)
```

### 6. Улучшенное определение тренда

**Проблема:** Одна EMA может давать ложные сигналы.

**Решение:** Использовать несколько таймфреймов для подтверждения тренда.

```mql5
// Добавить в TrendPullbackEA_Inputs.mqh
input bool   UseMultiTimeframe = true;  // Использовать мультитаймфрейм анализ
input ENUM_TIMEFRAMES TrendTF = PERIOD_H4; // Таймфрейм для определения тренда
```

### 7. Статистика торговли

**Проблема:** Нет отслеживания результатов торговли.

**Решение:** Добавить подсчет прибыли/убытков, количества сделок, статистики.

## 🛡️ Правила управления рисками

### Правило 1% (адаптировано для $5000):
- **Максимальный риск на сделку:** 0.5% = $25
- **Максимальный риск в день:** 1.5% = $75 (максимум 3 сделки)
- **Максимальный риск в неделю:** 3% = $150
- **Максимальная просадка:** 10% = $500 (стоп всей торговли)

### Правило соотношения риск/прибыль:
- **Минимум:** 2:1 (для консервативной торговли лучше 3:1)
- **Цель:** 3:1 или выше

### Правило максимальной просадки:
- При просадке 5% - уменьшить риск до 0.25%
- При просадке 7% - временно остановить торговлю
- При просадке 10% - полная остановка до анализа
